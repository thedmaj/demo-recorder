The most important stage. Opus generates the full demo script: 8–14 steps, narration (20–35 words each), visual state descriptions, Plaid Link phases, timing, and API response data.

// demo-script.json structure per step { id: "signal-insight", label: "Signal — ACH Risk Score", narration: "Plaid Signal scores the transaction…", durationMs: 10000, visualState: "Dark screen, teal gauge showing score 7…", interaction: { type: "wait", target: "signal-card" }, apiResponse: { score: 7, recommendation: "ACCEPT" }, plaidPhase: "" }
To modify: Edit demo-script.json directly and run --from=build. Change narration text, add/remove steps, adjust durationMs for timing. The dashboard narration editor does this visually.
Constraints: Narration must be 20–35 words per step. Signal scores 5–20 = ACCEPT. Never use "Trust Index". Include exactly one step with plaidPhase: "launch" for Plaid Link.

Generates a fully interactive single-page HTML demo app styled with the brand's design system. Each step is a data-testid="step-{id}" div. Also generates a Playwright automation script.

// DOM contract (enforced by validator) <div data-testid="step-{id}" class="step"> ... </div> window.goToStep = (id) => { /* show step div */ } window.getCurrentStep = () => { /* return active step id */ } // Each step has full-viewport layout, branded design, // data-testid on all interactive elements, // pre-populated API response data from demo-script.json

Extracts start/mid/end frames for each step, sends them to Sonnet Vision with the expected visualState from the script. Scores each step 0–100. Passes at ≥80.

for step in steps: frames = extractFrames(step.start, step.mid, step.end) score = sonnetVision(frames, step.visualState) if score < 80: failures.push({ step, score, frames }) if failures && iter < 3: triggerBuildRefinement(failures)
Threshold: Change QA_PASS_THRESHOLD in .env (default 80). View results in the dashboard QA tab — each step frame is shown with score.

6Vibe Coder Guide

The most common ways to modify the pipeline without deep code changes.

🎬 Change the demo story

Edit inputs/prompt.txt and run npm run demo. Include: product name, target persona, key features, tone. More specificity = better script. The prompt also supports a Brand URL for design extraction.

✏️ Tweak narration

Open the dashboard at localhost:4040 → Storyboard tab. Edit any step's narration inline and regenerate just that voice clip. Or edit demo-script.json and run --from=voiceover.

🎨 Change visual style

Edit scratch-app/index.html directly via the "Demo Apps" launcher in the dashboard. Select an element and describe the change. AI applies it in css/element mode (fast). Or run --from=build with updated brand JSON.

✂️ Trim a clip

Run npm run eval-recorder to open Remotion Studio at localhost:4001. Drag the trim handles on any scene. Or in the main Remotion Studio (localhost:3000), set startOffset / endOffset in frames (30fps).

📊 Add/edit overlays

Dashboard → "AI Overlay Suggestions" to see Gemini's suggestions. Or edit remotion-props.json directly. ZoomPunch, LowerThird, BadgeCallout, StatCounter are the available overlay types. Touchup stage does this via chat.

🔁 Re-record one step

Edit the step's playwright-script.json entry, then run --from=record. Set MANUAL_RECORD=true to drive the recording yourself. The dashboard shows step-level QA scores so you know which step failed.

🔊 Change the voice

Set ELEVENLABS_VOICE_ID in .env. Browse voices at localhost:4040/api/elevenlabs/voices. Then run --from=voiceover to regenerate all clips.

⏱ Fix audio/video sync

Dashboard → "Timing" tab shows per-step speed multipliers from sync-map.json. Drag sliders to adjust. Or manually edit sync-map.json and run --from=resync-audio then --from=render.

🤖 Understand a prompt

All LLM prompts live in scripts/scratch/utils/prompt-templates.js. Each function returns the exact text sent to the model. This is where DOM contract rules, brand voice guidelines, and Plaid Link instructions are defined.

8Model Selection Rationale

Why each model was chosen for each task.

Opus 4.7 Script & App Build

The two stages that require the most reasoning and creative output. Script generation must follow a complex narrative arc with brand voice rules. App build must produce valid DOM-contract HTML with full JS in one shot. Opus's larger context window and reasoning handle both reliably.

Sonnet 4.6 QA Vision, Research, Touchup

Vision tasks (QA frame scoring, Figma review) need strong multi-modal reasoning but don't require Opus-level creativity. Research benefits from Sonnet's balance of speed and quality for agentic tool-use loops. Touchup needs reliable instruction-following for props editing.

Haiku 4.5 Critique, Brand, PPT

Fast, cheap tasks: script quality-checking, brand JSON normalisation, PowerPoint slide generation. These are classification/extraction tasks where speed matters more than deep reasoning. Haiku is also used for the AI demo app editor in css/element mode for sub-3-second responses.

ElevenLabs Voice

eleven_multilingual_v2 at 192kbps/44.1kHz produces the most natural-sounding narration. stability: 0.75 prevents stutter artifacts that lower values introduce. Per-clip generation (not full concatenation) enables targeted regeneration of just the bad clips.

Gemini 2.0 Overlays

Overlay suggestion requires simultaneous analysis of video frames + structured demo data. Gemini 2.0 Flash's multimodal capabilities and native video understanding make it well-suited here. Gracefully skipped if Google credentials aren't available.

Playwright Recording

headless: false enables GPU compositor for capturing cross-origin iframes (real Plaid Link modal). CDP (Chrome DevTools Protocol) via frameLocator bypasses z-index for automating the Plaid SDK directly. devtools=true enables HiDPI recording at native 2× resolution.
