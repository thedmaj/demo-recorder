## File naming convention (critical)

The composition auto-discovers recordings by scanning `public/<compositionId>/` for files matching these prefixes. All files from the same recording share the same numeric timestamp suffix.

| Prefix | Source | Example |
|--------|--------|---------|
| `webcam` | Webcam video | `webcam1773678617807.mp4` |
| `display` | Screen capture (from display picker) | `display1773678617807.mp4` |
| `alternative1` | First extra source (screen-only recordings land here) | `alternative11773678617807.mp4` |
| `alternative2` | Second extra source | `alternative21773678617807.mp4` |
| `subs` | Auto-generated captions (Whisper.cpp) | `subs1773678617807.json` |

**CRITICAL**: The camera-matching logic (`get-camera.ts`) anchors on `webcam*` files. It finds `alternative1`, `display`, and `subs` files with the **same timestamp** as a `webcam` file. **If there is no `webcam*.mp4`, the scene will show "No recordings"** even if `alternative1*.mp4` exists.

### Fix for screen-only recordings (no webcam file)

When the user recorded screen-only, create a dummy silent webcam file with the same timestamp, OR rename the `alternative1` file to `webcam` with the same timestamp:

```bash
# Get the timestamp from the alternative1 file
# e.g. alternative11773678617807.mp4 → timestamp = 1773678617807

# Option A: rename alternative1 → webcam (makes it the "webcam" track showing the screen)
mv public/welcome/alternative11773678617807.mp4 public/welcome/webcam1773678617807.mp4

# Option B: symlink so both tracks exist
cp public/welcome/alternative11773678617807.mp4 public/welcome/webcam1773678617807.mp4
```

After renaming, the scene will show the screen recording in the webcam position.

## Composition structure (`Root.tsx`)

Three registered compositions:
- **`welcome`** — main editing composition; reads from `public/welcome/`
- **`record`** — "go to recorder" placeholder screen
- **`empty`** — blank starting point

To create a new composition for a different project, add a new `<Composition>` in `remotion/Root.tsx` with a new `id` (e.g. `"my-demo"`). Recordings saved to `public/my-demo/` will automatically appear in it.

## Scene types (props panel in Studio)

Edit via the **right-side props panel** in Remotion Studio or by editing `remotion/Root.tsx` defaultProps.

| Type | Purpose | Key props |
|------|---------|-----------|
| `videoscene` | Renders a recorded clip | `startOffset`, `endOffset`, `webcamPosition`, `bRolls`, `music` |
| `recorder` | "No recording yet" placeholder | `durationInFrames` |
| `title` | Title card | `title`, `subtitle`, `durationInFrames` |
| `endcard` | Outro with links | `channel`, `links`, `durationInFrames` |
| `tableofcontents` | Chapter list card | `durationInFrames` |

## Trimming clips

`startOffset` and `endOffset` are **frame counts** (FPS = 30):
- `startOffset: 30` → trim 1 second from the start
- `endOffset: 60` → trim 2 seconds from the end

Edit these in the Remotion Studio props panel (right side) while the `videoscene` is selected, or modify `defaultProps` in `remotion/Root.tsx`.

## Webcam position options

Controls where the webcam bubble is placed relative to the screen recording:

- `"top-left"`, `"top-right"`, `"bottom-left"`, `"bottom-right"` — corner pip
- `"center"` — full screen webcam
- `"previous"` — inherit from previous scene

## Canvas layout options (top-level prop)

- `"square"` — 1080×1080 (Twitter/LinkedIn)
- `"landscape"` — 1920×1080 (YouTube)
- `"portrait"` — 1080×1920 (TikTok/Reels)

Change via the `canvasLayout` prop in the Studio props panel.

## Captions / subtitles

Captions are generated by running Whisper.cpp:
```bash
cd /Users/dmajetic/Claude\ Test/our-recorder
~/.bun/bin/bun sub.ts
```
This downloads the model (~1.5GB on first run) and generates `subs<timestamp>.json` for each recording in `public/<compositionId>/`.

Captions appear as word-level overlays in the `videoscene`. Edit individual words directly in the Remotion Studio canvas (click a word to edit).

## B-roll

Add image or video overlays to a `videoscene` via the `bRolls` array:
```json
{
  "source": "sample-broll.jpg",
  "durationInFrames": 90,
  "from": 30
}
```
Can also be dragged into the scene in Studio (DragDropBRoll).

## Background music

Set `music` on any scene:
- `"epic"` — energetic
- `"calm"` — relaxed
- `"fun"` — upbeat
- `"previous"` — continue previous scene's music
- `"none"` — silence

## Adding a new composition for a new recording project

1. Add a new folder via the top-right dropdown in the recorder UI (e.g. `my-project`)
2. Record — files save to `public/my-project/`
3. Add to `remotion/Root.tsx`:

```tsx
<Composition
  component={Main}
  id="my-project"
  schema={videoConf}
  defaultProps={{
    theme: "light" as const,
    canvasLayout: "landscape" as const,
    platform: "youtube" as const,
    scenes: [
      {
        type: "videoscene" as const,
        webcamPosition: "bottom-right" as const,
        startOffset: 0,
        endOffset: 0,
        transitionToNextScene: false,
        newChapter: "",
        stopChapteringAfterThis: false,
        music: "none" as const,
        bRolls: [],
      },
    ],
    scenesAndMetadata: [],
  }}
  calculateMetadata={calcMetadata}
/>
```
