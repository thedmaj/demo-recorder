================================================================================
Upgrade — Personal Loans: up-funnel waterfall with Plaid Qualify → Layer + Consumer Report → Auth
================================================================================

**Host:** **Upgrade** — consumer lending / personal loans (near-prime installment lending)
**Canonical URL:** https://www.upgrade.com
**Brand URL:** https://www.upgrade.com

**User journey (one sentence):**
A personal-loan applicant is prequalified up-funnel with a friction-free Plaid Qualify check
(identity-only, no bank link), then connects their bank through a single Plaid Layer session so
Upgrade can pull a Plaid Consumer Report (Base Report, LendScore, Network Insights, Cash Flow
Insights) plus Auth — turning an early eligibility signal into an enhanced, funded offer.

**Story arc (short prose):**
Jordan Rivera applies for a personal loan on Upgrade. Before asking for anything but identity,
Upgrade runs **Plaid Qualify** — a friction-free check that returns an early LendScore and an
eligibility signal from verified PII alone, no bank connection. Jordan is prequalified for a
starter offer. To unlock better terms, Upgrade invites Jordan to connect a bank: a single
**Plaid Layer** session confirms identity and links the primary checking account in one step.
Behind the scenes Upgrade retrieves the Layer session (verified identity + an Auth-enabled
access token), pulls ACH routing/account numbers, and generates a **Plaid Consumer Report**.
When the report-ready webhook fires, Upgrade reads bank-verified income, cash-flow stability,
and Network Insights (rent and BNPL repayment) — and a higher-accuracy post-bank LendScore.
Jordan's offer is enhanced — a larger loan at a better rate — and accepted; funds are disbursed
straight to the verified account. One waterfall: prequalify instantly, enhance with verified
bank data, approve and fund.

**Product-specific reminders:**
- **Qualify is server-side, NO modal, NO bank link** — never show a Link/Layer modal in beats 1–2.
  `bureau_report` is `null` (bureau integration not live) — do NOT show bureau data.
- **LendScore scale — UNRESOLVED:** the approved `cra-lend-score` KB says 1–99 (higher = safer);
  the Qualify spec referenced 1–850. **Keep on-screen LendScore consistent with the approved
  `cra-lend-score` KB (1–99, higher = safer)** and do NOT assert a Qualify-specific range until
  confirmed. LendScore direction is the OPPOSITE of Signal. Show the post-bank LendScore (beat 6)
  as higher-confidence than the pre-bank Qualify score (beat 1).
- **Network Insights is post-bank only** — appears in beat 6, never in the Qualify beats. V0
  signals are rent (Transfer-based) + BNPL (network partners); do NOT show cash-advance/EWA
  signals under Network Insights (out of V0 scope).
- **CRA:** real Plaid session (Layer launch); Consumer Report generated SERVER-SIDE with the async
  `USER_CHECK_REPORT_READY` beat. No second CRA Link launch.
- **Layer:** pre-step narration ends on the button; Layer step narrates INSIDE the modal only.
- **Auth:** account/routing from `/auth/get` using the `access_token` from `/user_account/session/get`.
- Realistic-but-idealized data: no 100/100 anything, no sub-1s API claims, no fabricated reason codes.

**Compliance / user data (one line):**
CRA permissible purpose is attested at `/user/create`; Qualify uses `pull_type:"none"` (no bureau
inquiry); identity is carried by Qualify PII + the Layer session — FCRA-grade Consumer Report.
POC conversion/offer-rate/dollar targets are HELD pending human sign-off — qualitative framing only.
