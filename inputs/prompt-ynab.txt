A digital-native young professional links financial accounts in YNAB to view spending, investments, and liabilities in one unified money dashboard.

**Story arc (short prose):**  
Start with YNAB's core budgeting promise ("give every dollar a job"), then show how synced transaction data keeps the budget current without manual cleanup. **Research-informed value proposition talking points (use in narration + final summary slide):**  
- "From fragmented account data to one connected financial picture: cash flow, investments, and liabilities in one place."  
- "Transactions Sync keeps the budget current with clean merchant/category detail, so users spend less time reconciling and more time deciding."  
- "Investments + liabilities turn a budgeting app into a full financial operating system with net-worth context."  
- "Use debt structure signals like APR, due date, and balance trend to plan realistic payoff priorities."  
- "Position the user outcome as proactive guidance ("what should I do next?"), not passive tracking."

**Product-specific reminders** (delete if N/A):  
- Link: exactly one pre-Link host screen and exactly one `plaidPhase: "launch"` step.  
- Transactions: emphasize merchant name/logo, category (primary + detailed if present), amount, date, pending vs posted, and sync updates (added/modified/removed).  
- Investments: emphasize account name/type, holdings ticker symbol, quantity, current price, and market value.  
- Liabilities: emphasize mortgage APR, due date, minimum/payment amount, and opening vs current balance deltas.
