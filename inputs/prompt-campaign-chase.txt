Chase Bank Online Transfers — Fund Instantly, Verify Ownership, Prevent Fraud
================================================================================

Host: Chase Bank — U.S. retail banking / Banking segment
Canonical URL: https://www.chase.com
Brand URL: https://www.chase.com

User journey (one sentence):
A Chase customer adds an external bank account and initiates a transfer using
standard Plaid Link to verify account ownership with Identity Match, retrieve
ACH details with Auth, and risk-score the transaction with Signal — converting
funding attempts while blocking fraud.

Story arc (short prose):
Chase wants to grow inbound ACH funding volumes without absorbing return risk
or building friction that kills conversion. Today, account-funding flows ask
customers to wait days for micro-deposit confirmation or abandon the flow
entirely. Plaid enters at the "Add External Account" CTA on the Chase Transfers
screen. Standard Plaid Link connects the external bank in seconds. Identity
Match confirms the account belongs to the Chase customer on record — not a
fraudster using stolen credentials. Auth retrieves verified account and routing
numbers instantly. The customer enters the transfer amount, and Signal Evaluate
risk-scores that specific transaction. Low-risk users are approved and funded
immediately; Chase releases funds same-session without a micro-deposit wait.
The result: higher conversion, higher funding rates, and a strong fraud gate —
all from a single Plaid integration.

MANDATORY FLOW ORDER (STORYBOARD + NARRATION)
-------------------------------------------------------------------------------

| # | Beat type   | What the viewer sees                                                        | Key API / UX                              |
|---|-------------|-----------------------------------------------------------------------------|-------------------------------------------|
| 1 | host        | Chase Transfers screen — account list, balance, CTA to add external account | Entry point: "Add External Account" btn   |

| 4 | host        | External account connected — Identity Match result shown                    | `POST /identity/match` — per-attr scores  |
| 5 | host        | Auth confirmation — ACH account + routing numbers retrieved                 | `POST /auth/get` — numbers + institution  |
| 6 | host        | Enter transfer amount + destination account                                 | User input, prefilled sender/receiver     |

| 8 | host        | Transfer approved — funds available today                                   | Outcome: confirmation + availability      |

Hard rules:
- The Plaid Link step is a single `"plaidPhase": "launch"` step — no sub-steps.
- Pre-Link step narration MUST end with the action that opens the modal
  (e.g. "...she taps Add External Account.").
- Plaid Link step narration describes content VISIBLE INSIDE the modal only —
  never the act of opening it (per CLAUDE.md Plaid Link narration boundary rule).
- Identity Match scores: realistic values (name 94, address 91, phone 88).
  Do NOT use 100/100. Show per-attribute score breakdown.
- Signal score: realistic low-risk ACCEPT value (consumer_initiated: 12,
  bank_initiated: 7). Higher score = higher ACH return risk. ACCEPT = low score.
  Do NOT use "Trust Index" — use "ACH return risk score" in narration.
- Signal ACCEPT result must be fully resolved on screen before advancing —
  never leave a spinner in the final frame of the signal step.
- After Plaid onSuccess fires, use `window._plaidInstitutionName`,
  `window._plaidAccountName`, `window._plaidAccountMask` for display labels.
  Never hardcode bank names in HTML.

Compliance-safe wording:
- Prefer "can help," "supports," "designed to," "reduces risk of."

- Signal ACCEPT = low return risk indication, not a guarantee funds will clear.

API RESPONSE WIRING (for build agent accuracy)
-------------------------------------------------------------------------------

Step 4 — Identity Match:
- Endpoint: `POST /identity/match`
- Show per-attribute match scores (name, address, phone_number)
- Status: `"status": "MATCH"` or equivalent success state
- Panel: visible, JSON collapsed by default

Step 5 — Auth:
- Endpoint: `POST /auth/get`
- Show account number (masked), routing number, account type
- Institution name from `window._plaidInstitutionName`
- Panel: visible, JSON collapsed by default

Step 7 — Signal:
- Endpoint: `POST /signal/evaluate`
- `scores.customer_initiated_risk_level.score`: 12
- `scores.bank_initiated_risk_level.score`: 7
- `core_attributes.days_since_first_plaid_connection`: realistic (e.g. 412)
- `request_decision`: `"APPROVE"`
- Animation: show 3 check rows resolving sequentially, then ACCEPT badge
- Panel: visible, JSON collapsed by default
- CRITICAL: ACCEPT badge must be fully visible in the final frame of the step
