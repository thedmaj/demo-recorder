MANDATORY FLOW ORDER (STORYBOARD + NARRATION + goToStep MUST ALIGN)
-------------------------------------------------------------------------------

| # | Beat type | What happens | Key API or UX |
|---|-----------|--------------|----------------|
| 1 | host | Claims agent/policyholder selects digital bank payout for approved claim | claim disbursement setup |
| 2 | host | User taps "Link payout account" | launches Plaid Link |
| 3 | Link | User completes Link institution + account selection | Plaid Link modal |
| 4 | host | Token exchange confirms linked account metadata in host UI | `/item/public_token/exchange` |
| 5 | Plaid insight | Ownership verification against claimant profile | `POST /identity/match` |
| 6 | Plaid insight | ACH rails retrieval for selected account | `POST /auth/get` |
| 7 | host | Liberty confirms payout destination and amount, then submits | claim payout confirmation |
| 8 | host | Success outcome: payout initiated with verified account details | clear CTA/outcome |

Hard rules:
- Identity Match appears before Auth in the story.
- Do not narrate guaranteed fraud elimination.
- Keep API endpoint naming exact (`POST /identity/match`, `POST /auth/get`).

Async lifecycle/readiness:
- This flow is synchronous for demo purposes.
- Show lightweight processing states, then immediate verification outcome.

Primary fetch endpoints after Link:
- `POST /identity/match`
- `POST /auth/get`
