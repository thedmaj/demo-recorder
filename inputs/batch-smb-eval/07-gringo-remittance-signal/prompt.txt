================================================================================
Gringo Coin — Lower-risk cross-border payments to Brazil (Plaid IDV + Auth + Signal)
================================================================================

**Host:** **Gringo Coin** — cross-border / P2P-international payments (SMB; US → Brazil)

**User journey (one sentence):**
A US sender verifies identity with Plaid Identity Verification, links a bank with Plaid Link, and Gringo Coin pulls the ACH rails (Auth) and risk-scores the funding debit with Plaid Signal before releasing a cross-border payment to Brazil.

**Story arc (short prose):**
Daniela Costa wants to send money from her US bank to family in Brazil. Gringo Coin needs to know the sender is real and that the ACH pull won't bounce or be fraudulent. Daniela completes a Plaid IDV session for KYC, then links her bank in one Plaid Link tap. Gringo Coin retrieves verified account/routing (Auth) and runs Plaid Signal on the specific debit — a low return-risk score returns ACCEPT, so the payment to Brazil clears without holds.

STORYBOARD BEATS (ORDER = SCRIPT ORDER)
-------------------------------------------------------------------------------

| # | Beat | What the viewer sees | API sequence | Reveal / CTA |
|---|------|----------------------|--------------|--------------|
| 1 | host | Gringo Coin app — "Send money to Brazil." Verify identity to continue. Ends on tap. | `POST /identity_verification create` | `Verify identity` |
| 2 | link (IDV) | Live Plaid IDV session — document + data source + selfie. | `onSuccess` → `/identity_verification/get` → `success` | Identity verified |
| 3 | host | "Link your bank to fund the transfer." Ends on tap. | `POST /link/token/create` (`["auth"]`) | `Link your bank` |
| 4 | link (Auth) | Live Plaid Link — connect bank, choose checking. | `onSuccess` → `/item/public_token/exchange`; `/auth/get` | Account linked |
| 5 | host | Enter amount — **$300.00** to Brazil (≈ R$). | (amount captured) | `Review transfer` |
| 6 | signal-insight | Plaid insight panel: **Signal** scores this debit — `customer_initiated_return_risk.score` **14**, `bank_initiated_return_risk.score` **11**, `ruleset.result` **ACCEPT**. | `POST /signal/evaluate` | "Low return risk → ACCEPT" |
| 7 | host | **Sent.** Payment released to Brazil; no hold. | (transfer released) | `Send` |

**Product-specific reminders:**
- Signal score is 1–99, higher = HIGHER ACH return risk; ACCEPT demos use LOW scores (5–20), never 80s/90s. `ruleset.result` ∈ {ACCEPT, REVIEW, REROUTE}; happy path ACCEPT. Never label Signal scores "Trust Index".
- IDV statuses: show `success`; never API errors.
- Auth account mask/institution from Link `onSuccess` metadata only.
- Happy path only.

**Compliance / user data:** Standard KYC via Plaid IDV; representative sandbox persona (Leslie Knope).
