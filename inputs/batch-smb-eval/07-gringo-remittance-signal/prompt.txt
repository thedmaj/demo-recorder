================================================================================
Gringo Coin — Lower-risk cross-border payments to Brazil (Identity Verification + Plaid Auth + Plaid Signal)
================================================================================

**Host:** **Gringo Coin** — cross-border / P2P-international payments (SMB; US → Brazil)

**User journey (one sentence):**
A US sender completes a real Plaid Identity Verification session for KYC, then connects their bank in a second real Plaid Link session, and Gringo Coin pulls the ACH rails (Plaid Auth) and risk-scores the funding debit with Plaid Signal before releasing a cross-border payment to Brazil.

**Story arc (short prose):**
Daniela Costa wants to send money from her US bank to family in Brazil. Gringo Coin needs to know the sender is real and that the ACH pull won't bounce or be fraudulent. Daniela first completes a real Plaid Identity Verification session (document + selfie). Then she opens a second real Plaid Link session to connect her bank. Gringo Coin retrieves verified account/routing (Plaid Auth) and runs Plaid Signal on the specific debit — a low return-risk score returns ACCEPT, so the payment to Brazil clears without holds.

**Product-specific reminders:**
- Two real launch steps (IDV, then bank Auth) — each its own session/token; never combine identity_verification with auth in one token.
- Signal score is 1–99, higher = HIGHER ACH return risk; ACCEPT demos use LOW scores (5–20). `ruleset.result` ∈ {ACCEPT, REVIEW, REROUTE}; happy path ACCEPT. Never label Signal scores "Trust Index".
- IDV statuses: show verified/`success`; never API errors. Plaid Auth account mask/institution from Link `onSuccess` metadata only.
- Happy path only.

**Compliance / user data:** Standard KYC via real Plaid Identity Verification; representative sandbox persona (Leslie Knope).
