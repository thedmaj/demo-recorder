================================================================================
Swype — Fund a stablecoin wallet from any US bank (Plaid IDV + Auth + Identity Match)
================================================================================

**Host:** **Swype** — crypto / blockchain-to-fiat wallet (SMB; stablecoin funding)
**Brand URL:** https://www.swype.network

**User journey (one sentence):**
A new Swype user verifies identity with Plaid IDV, links a US bank with Plaid Link, and Swype confirms the linked account belongs to them (Identity Match) and pulls the ACH rails (Auth) before funding their stablecoin wallet with a verified ACH transfer.

**Story arc (short prose):**
Alex Rivera wants to fund a stablecoin wallet from a US checking account. Swype needs one-time KYC and assurance the funding account is really theirs — without high-friction onboarding. Alex completes a Plaid IDV session for KYC, then links the bank in one Plaid Link tap. Behind the scenes Swype confirms the account name matches Alex (Identity Match) and retrieves account + routing rails (Auth), then funds the wallet via ACH. Low friction, low fraud, fast funding.

STORYBOARD BEATS (ORDER = SCRIPT ORDER)
-------------------------------------------------------------------------------

| # | Beat | What the viewer sees | API sequence | Reveal / CTA |
|---|------|----------------------|--------------|--------------|
| 1 | host | Swype app — "Fund your wallet." New user prompted to verify identity. Ends on tap. | `POST /identity_verification create` | `Verify identity` |
| 2 | link (IDV) | Live Plaid IDV session — document + data source + selfie. | `onSuccess` → `/identity_verification/get` → `success` | Identity verified |
| 3 | host | "Link your bank to fund your wallet." Ends on tap. | `POST /link/token/create` (`["auth","identity"]`) | `Link your bank` |
| 4 | link (Auth) | Live Plaid Link — connect bank, choose checking. | `onSuccess` → `/item/public_token/exchange` | Account linked |
| 5 | identity-match-insight | Plaid insight panel: linked account name matches the verified user — `legal_name.score` **96**. | `POST /identity/match` | "Ownership confirmed" |
| 6 | auth-insight | Plaid insight panel: ACH rails retrieved (checking ****8842; account + routing). | `POST /auth/get` | "ACH rails ready" |
| 7 | host | **Funded.** $250.00 moved into the stablecoin wallet via verified ACH. | (funding initiated) | `Fund wallet` |

**Product-specific reminders:**
- Identity Match: scores 0–100; pass the verified name as `legal_name`; ~70+ is a strong match. Never label scores "Trust Index".
- IDV statuses: show `success`; never API errors.
- Auth account mask/institution from Link `onSuccess` metadata only.
- Happy path only.
