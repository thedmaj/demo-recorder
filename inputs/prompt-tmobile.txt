**User journey (one sentence):**
A T-Mobile applicant who is thin-file applies for phone financing, consents to secure data sharing through Plaid Link, and receives a second-look underwriting decision powered by CRA and income insights.

**Story arc (short prose):**
Open on T-Mobile device checkout where a customer chooses installment financing. Show that legacy underwriting can miss qualified thin-file applicants. Introduce Plaid-powered second look: user consents once, T-Mobile retrieves CRA base report and income insights, then returns an explainable approve outcome with terms.

INFERRED API STACK (IMPLEMENTATION INTENT)
-------------------------------------------------------------------------------

Use these endpoints and sequencing for this use case:
1. `POST /user/create` (or account-compatible identity payload path)
2. `POST /link/token/create` with CRA products:
   - `products: ['cra_base_report', 'cra_income_insights']`
   - `consumer_report_permissible_purpose: 'EXTENSION_OF_CREDIT'`
   - `cra_options.days_requested` as appropriate
3. `POST /cra/check_report/base_report/get`
4. `POST /cra/check_report/income_insights/get`

If account configuration is CRA base-report-only, still keep storyboard language for second-look underwriting but adapt insight slide to available report outputs.
